# Pluggable Storage Backend Interface Design

> **Issue #13** — Design for clean storage backend abstraction.

## Problem Statement

Currently, the storage layer has SQLiteStorage and PostgresStorage, but there's no clean abstract interface. Contributors can't easily add Turso (edge), Redis (ephemeral), or S3 (archival) backends without touching core logic.

## Current Architecture

```
engine.py
    │
    ▼
BaseStorage (abstract)
    │
    ├── SQLiteStorage (local mode)
    └── PostgresStorage (team mode)
```

## Proposed Interface

```python
from abc import ABC, abstractmethod
from typing import Any
from datetime import datetime

class StorageBackend(ABC):
    """Abstract interface for Engram storage implementations."""
    
    # Lifecycle
    async def connect(self) -> None: ...
    async def close(self) -> None: ...
    
    # Fact operations
    async def insert_fact(self, fact: dict[str, Any]) -> int: ...
    async def get_fact_by_id(self, fact_id: str) -> dict | None: ...
    async def get_facts_by_ids(self, ids: list[str]) -> dict[str, dict]: ...
    async def get_current_facts_in_scope(
        self, scope: str | None = None, fact_type: str | None = None,
        as_of: str | None = None, limit: int = 200, include_ephemeral: bool = False
    ) -> list[dict]: ...
    async def find_duplicate(self, content_hash: str, scope: str) -> str | None: ...
    async def close_validity_window(self, *, lineage_id: str | None = None, fact_id: str | None = None) -> None: ...
    
    # Query operations  
    async def fts_search(self, query: str, limit: int = 20) -> list[int]: ...
    async def get_facts_by_rowids(self, rowids: list[int]) -> list[dict]: ...
    async def get_facts_by_embedding_model(self, model: str, limit: int = 1000) -> list[dict]: ...
    async def get_facts_since(self, since: str, limit: int = 200) -> list[dict]: ...
    async def get_facts_by_lineage(self, lineage_id: str) -> list[dict]: ...
    
    # Conflict operations
    async def insert_conflict(self, conflict: dict[str, Any]) -> None: ...
    async def get_conflicts(self, scope: str | None = None, status: str = "open") -> list[dict]: ...
    async def resolve_conflict(self, conflict_id: str, resolution: str, resolution_type: str | None = None) -> None: ...
    async def conflict_exists(self, fact_a_id: str, fact_b_id: str) -> bool: ...
    async def get_conflicting_fact_ids(self, fact_id: str) -> set[str]: ...
    
    # Agent operations
    async def register_agent(self, agent_id: str, engineer: str, label: str | None = None) -> None: ...
    async def get_agents(self) -> list[dict]: ...
    async def get_agent(self, agent_id: str) -> dict | None: ...
    
    # Durability operations
    async def promote_fact(self, fact_id: str) -> bool: ...
    async def increment_query_hits(self, fact_ids: list[str]) -> None: ...
    async def get_promotable_ephemeral_facts(self, min_hits: int = 2) -> list[dict]: ...
    async def get_facts_by_durability(self, durability: str) -> list[dict]: ...
    async def retire_stale_facts(self) -> int: ...
    
    # Webhook operations
    async def insert_webhook(self, webhook: dict[str, Any]) -> None: ...
    async def get_webhooks(self) -> list[dict]: ...
    async def get_webhook_by_id(self, webhook_id: str) -> dict | None: ...
    async def delete_webhook(self, webhook_id: str) -> bool: ...
    async def queue_webhook_delivery(self, delivery: dict[str, Any]) -> None: ...
    
    # Utility
    async def get_stats(self) -> dict[str, Any]: ...
    async def get_distinct_scopes(self) -> list[str]: ...
    async def expire_ttl_facts(self) -> int: ...
    async def get_expiring_facts(self, days_ahead: int = 7) -> list[dict]: ...
```

## Supported Backends (Future)

| Backend | Use Case | Status |
|---------|----------|--------|
| SQLiteStorage | Local dev, self-hosted | ✅ Implemented |
| PostgresStorage | Team mode | ✅ Implemented |
| TursoStorage | Edge deployments | Planned |
| RedisStorage | Ephemeral/cache layer | Planned |
| S3Storage | Archive cold data | Planned |

## Implementation

1. **Formalize BaseStorage** — Already exists in `storage.py`, but document as the official interface
2. **Add backend capability check** — Some backends may not support all features (e.g., FTS, embeddings)
3. **Configuration-driven backend selection** — `ENGRAM_STORAGE_BACKEND=sqlite|postgres|turso`

## Risk Assessment

- **Breaking changes**: Low — existing implementations already match
- **Testing**: Need integration tests per backend
- **Performance variance**: Different backends have different performance profiles
